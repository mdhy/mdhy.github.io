Hyper-V + CentOS5 + Promiscuous_mode, その後

Sep 6th, 2011

　Hyper-V って promiscuous_mode 使えない！？ ずーと Hyper-V にはプロミスキャスモードがないものと思ってましたが、Hyper-V 2.0 から機能が追加されていたようです。 MCT の憂鬱 – Hyper-V 2.0 新機能～ネットワーク編その１： http://naonao71.wordpress.com/2009/09/01/hyper-v-2-0-%E6%96%B0%E6%A9%9F%E8%83%BD%EF%BD%9E%E3%83%8D%E3%83%83%E3%83%88%E3%83%AF%E3%83%BC%E3%82%AF%E7%B7%A8%E3%81%9D%E3%81%AE%EF%BC%91/ > ネットワークの設定において「MACアドレスのスプーフィングを有効にする」という機能が追加されました。 うん、よろしい。 Hyper-V 2.0 on Hyper-V Server 2008 R2 な環境では，ゲストの仮想NICをプロミスキャスモードにできない?： http://d.hatena.ne.jp/ka_maru/20091119/1258645882 > Virtual Server 2005あたりでは，XMLファイルを弄れば行けた模様．が，上記リンク先の最新2POST（lemon75とManuel Godoyのもの）を読む限り多分ムリ．トドメを刺したい人は，試しにwiresharkでも走らせてm9(^Д^)ﾌﾟｷﾞｬｰしてください． 期待とうらはらにできないという見解もあるもよう。 ###　どっちなの？！ かくいう私も最近までできないと思い込んでたけど、結論からというと CentOS でいうと Linux Integration Services v2.1 との組み合わせてでプロミスキャス・モードでけた。 Hyper-V には NIC が2種類ある「レガシー・ネットワーク・アダプタ」「統合サービス・ネットワーク・アダプタ」がある。 「レガシー・ネットワーク・アダプタ」については「MACアドレスのスプーフィングを有効にする」だけでプロミスキャス・モードが使える。 「統合サービス・ネットワーク・アダプタ」はちょっとくせがあった。 【Linux Integration Services v2.1 Read Me.pdf】 の15ページには > ＃ rmmod netvsc.ko > ＃ modprobe netvsc promisc_mode=1 と書いてあるが、起動時には有効になっていない。 どうも /etc/modprobe.conf に options 設定を書いても有効にならない もうこうなったら initrd に手を加えるしかない！！！ 結果、initrd を展開した init 内の > insmod netvsc.ko を > insmod netvsc.ko promisc_mode=1 に書き換えて initrd を纏めて /boot にコピーして再起動してはい、出来上がり。
