ＩＰｖ６の話

Jun 16th, 2009

またまたＩＰｖ６トンネリングを設定しました。

前に設定した時はＨＥＸＡＧＯのＧａｔｅｗａｙ６Ｃｌｉｅｎｔ（ http://go6.net/ ）を使っていました。

海を越えるため国内のＩＰｖ６サイトのＩＰアドレスにＰｉｎｇを通すと４００ｍｓ超という残念な結果でした。

遅いうえに安定せず、ＬｉｎｕｘではＮＡＴを越せないとなっては使い道がなくなり放置状態でしたｗ

＝＝＝＝＝＝

数日まえ思い出したように「ＫＡＭＥ」（ http://www.kame.net/ ）を踊らせたくなり、ＨＥＸＡＧＯ以外のサービスを探していると、Ｆｅｅｌ６( http://www.feel6.jp/ )という無料のＩＰｖ６トンネリングサービスを見つけました。

dtcp( http://www.imasy.or.jp/~ume/dtcp/dtcp.html )という技術を使ってトンネルを張るようです。

これを使うにはdtcpクライアントが必要にるようです。

Ｃｅｎｔｏｓ５で使ってみようと思っているので「ＫＡＭＥ」ではなく、「ＵＳＡＧＩ」（ http://www.linux-ipv6.org/ ）のdtcpクライアントを使うことにしました。

手順は以下参照

http://blog.goo.ne.jp/kai881/e/702c3fdd012fbeb298eb5601750bd32c

http://www.tugumi.net/tips/feel6linux/

＝＝＝＝＝＝

ping6 www.feel6.jp

の結果は平均８ｍｓくらいで落ち着いています。

加えてradvd ( http://www.atmarkit.co.jp/flinux/special/usagi/usagi03b.html ) を動かせば、ホームネットワークをＩＰｖ６化できるオマケ付です♪

そしてこのＷＥＢサーバもデュアルスタック（IPv4/IPv6の両方からアクセスできる）にしてみました。

どちらでアクセスしているかを上の

「あなたのＩＰアドレスは => ○○○○ (IPv○で提供中)」

で解りやすく表示しています。

ぜひ試してみて下さい。

ある日のSSLラッパー

Jun 9th, 2009

よー！ちぇげらっ！！のラッパーではありません。

ＳＳＬ通信機能を持たないＷＥＢサーバとクライアントの間において、クライアントとの通信をＷＥＢサーバの直前まで暗号化を可能にするものです。

解りにくい説明になりましたが、少し噛み砕くとインターネットからの受け口が直接ＷＥＢサーバだったものをＳＳＬラッパーを受け口にして、かつ”リバースプロクシ”として動作さてＷＥＢサーバと通信させるというものです。

＝＝＝＝＝

今なぜＳＳＬ（ラッパー）なのかというと、WEBでのセキュリティに興味でたという理由からです。

試すだけなら自家製の認証局を準備すればいいんですが実用的ではないので、第三者機関が発行する証明書を利用することにしました。

お金をはらえば当たり前の証明書を手に入れることができるのですが、半分遊びとなると大金をだすきはありません。

そうなると「無料で」という選択になります。無料だと有料にくらべて何が劣っているかというと「信頼されていない」ということです。

今回の場合は、送信する内容を暗号化したいだけなので「信頼されていなく」てもかまいませんので、「無料」でいくことにしました。

＝＝＝＝＝

グーグル先生に「SSL証明書無料」を聞いてみたところ「StartSSL」と「CAcert」を見つけました。

「CAcert」は以前使っていましたが、パスワードを忘れてしまったので、「StartSSL」にしました。

StartSSL ( http://www.startssl.com/ )

取得手順とか設定とかはグーグル先生に聞いて下さい。

とりあえずサーバに埋め込んでみました。

https://www.medanohaya.net/

＝＝＝＝＝

無料ということでメジャーなブラウザには、ルート証明書も中間証明書もはいていないため、インストールをする必要があります。

http://www.startssl.com/certs/

「ca.crt」と「sub.class1.server.ca.crt」を入れれば、見ることができますｗ

Hyper-V Server 2008 R2を試してみる２

May 23rd, 2009

Hyper-V Server 2008 R2を使ってみて決心がつきました。

CentOS+Xenの環境からHyper-V Server 2008 R2の環境へ移行します。

Windowsはだめだ！！なんて叫ぶ人もいますが、やはり世界一は世界一の扱いやすさと世界一の大胆さがあります。

今の環境であるCentOS+Xenでは、ディスクI/Oとネットークあたりが調子がおかしいです。

チューニングは全く施さずに使っていたわけですが、使えるので不満はそれほどありませんでした。

（ディスクアクセスが遅いのは、ソフトウェアRAIDを使っているからでしょう。iSCSIとかにしたいんですが、PC３台構成になるのが悩ましいです。）

Hyper-Vに関しては、とても安定しています（当たり前かｗ）

各部はブラックボックス化しているため触ろうとして触れないのですが、触る必要もないでしょう。

Windows製品であることから、基本はWindowsの親戚しか動かすことしかできないようです。

Suseエンプラはサポートしているようですが、Redhatエンプラもサポートが決まったようです。

リリースノートにはRedhatエンプラの名はありませんが、どうもRedhatエンプラにもネットワークが最適化される統合ソフトウェアがインストールできるようです。

Redhatエンプラに入るということは、CentOSにも入ります♪

＝＝＝＝＝＝

Hyper-V Server 2008 R2 は何にも入っていませんが、ファイル共有やリモートデスクトップが使えるのが助かります。

Windows7 RCが無償で使えて、Hyper-Vマネージャを利用できるのでホームユースでは十分です。

Hyper-V Server 2008 R2を試してみる

May 22nd, 2009

先日かいしゃの面談がありました。

もっとしっかりアピールするようにだそうです。

まったくそのとおりだとおもいましたｗ

＝＝＝＝＝＝＝＝

広く深く（リナックスもウインドウズも）知識を身につけては？ということでした。

私はネットワークエンジニア志望ですよ。そんなアバウトなーなんて思いながら、無償で利用できる「Windows Server 2008」の機能限定版である「Hyper-V Server 2008 R2 RC」をダウソして眠っていたSC440にインスコしてみました。

＝＝＝＝＝＝＝＝

インストールは簡単そのもの。

CDを入れてブートして、言語とキーボードを選択してOKをポチポチで入ります。

パスワードを設定して、ログインすると・・・コマンドプロンプトが２つしか動いていません。

そう、Server Core だけでシェル（Explorer）は入っていませんｗ

しかし臆することはないんです！！

ほとんどの操作がmmcを使ってリモート管理できます。

Hyper-Vに関しては、Hyper-Vマネージャーを使えば問題なしです。

＝＝＝＝＝＝＝

Windows 7 RCが評価版で手に入るので、mmcやらHyper-Vマネージャーは直ぐにセットアップできて、使えるようになりました。

Windows Server 2008といってもWindowsです。

IEはインストールできませんが、Firefoxならインストールできます♪

Firefoxも64ビット対応を入れないと日本語変換が使えませんが、入れれば普通に使えますｗ

＝＝＝＝＝＝＝

Windows 7はVMware Playerで動かします。

こいつほんとに良くできています♪

＝＝＝＝＝＝＝

今回わかったこと

Server Coreだけでも、ネットワーク周りとディスク周りはもちろん「サーバ」用なので、Server2008で動くソフトウェアを入れてやれば、ただで安定した○○サーバが作れるようです。（が使ってはいけない。）

とても興味が湧きましたよ！！

＝＝＝＝＝＝＝＝

こうなると今の３台の仮想環境をXenからHyper-Vに移し変える日も遠くなさそうです。

Macbookのキーボードを交換する

May 13th, 2009

数日前にYahooオークションで初期のMacbookのキーボード（JIS配列）を二枚落札しました。

送料あわせて３７５０円。

いままではUS配を使っていましたが、ビールをぶっかけたのと黄色くなったので、新しいキーボードを検討していましたが新品は１５０００円くらいと高いんです。

ビールぶっかけて余命ん？ヶ月かわからないこれに１５０００円も追加投資するなら違うものを買います＞＜

しかしながら、US配列キーボードがなかなか出品されていので、今後誰かに譲るかもしれない？ことを考えると、日本語かな？という脳内補完を行ってJIS配列を落札しました。

＝＝＝＝＝＝

取り付けは付いているUS配列のキーボードを取り外して、取り付ければ自動認識してくれます。

以外とJIS配列は使いやすいです♪

ジャンクだったため、少し黄ばんでいるので強気でお手入れしてみます。
