じぶん銀行の口座開設しようと思います

第一生命が「第一生命保険株式会社」ではなくて「第一生命保険相互会社」であることを「株式会社化・上場に関する方針のお知らせ」というDMで初めて知った林です。

こんばんは。

某大手に常駐していたときに三井住友生命の営業人が生命保険の加入を渋る私に「生命保険はあきらめました。年金保険はどうですか」という言葉を思い出して、「定期預金」を積み立ててみようと思い立ちました。

タンス貯金（銀行に預金しておく）でもいいのですが、話のネタの一つとして「定期預金」を始めてみようと思います。

まず、ジャパンネット銀行に１万円/月。そして、じぶん銀行に１万円/月。

「じぶん銀行」はついさっき見つけました。

参考リンク：じぶん銀行（http://www.jibunbank.co.jp/%EF%BC%89

会社の給与振込み先が東京三菱UFJ銀行を指定なので、東京三菱UFJ銀行で積み立てをしようと思ったのですが、金利が低いｗ

それでAU使ってるし、と東京三菱UFJ銀行使ってるしで「じぶん銀行」というわけです。

ジャパンネット銀行の金利0.8％/年、じぶん銀行の金利0.6％/年。

東京三菱UFJ銀行のスーパー定期預金の金利0.35％/年ｗｗ

現在の定期預金残高2万円ｗｗｗ

5年くらい続けたら定期預金残高120万円。金利分9600円（0.8％で計算した場合）ｗｗｗｗ

ローカルネットNicoCache始動

Jul 14th, 2008

世の中には便利なものがたくさんあります♪

みなさんもご存知のとおり日本を代表する動画共有サイトであるニコニコ動画！！

このニコニコ動画は動画配信サイトであることから広帯域なインターネット回線が必要となってきます。

毎日違った動画を見ているという使い方もありますが、同じ動画を見に行くという使い方も多いと思います。

一日一回にしてもブラウザの設定や性格である一定期間が過ぎると動画の内容が違っているのに再読み込みをしようとがんばる時があります。

これがネットが混みそうな時間になるとイライラしくるワケですｗ

そこで登場したのが「NicoCahce」というわけです。

参考URL：http://homepage1.nifty.com/asr/tools/nicocache.html

PC上でキャッシュサーバとして動作してニコニコ動画の動画のみをPCの中に保存しつづけます。

読み込みが完了した動画は次回からはPC内のファイルを読み込むためにニコニコ動画やインターネット回線の事情に影響されないくなります。

この「NicoCache」は便利なことにJavaで書かれていますが、セキュリティを確保するために動画データが保存されいるPCのブラウザからしか見ることできません＞＜

不便です。これではリソースをうまく配分できません。

ので、すべてからアクセスできるように改造しましたｗ

プログラムとニラメッコして１分。製作者のすばらしいコーディング力のおかげで該当箇所を発見！！

何も考えずに平書きで「0.0.0.0」にするわけですｗコンパイルして、jarでまとめで完了。

実行してみると正常に動作している模様♪

というわけで、ローカルネットNicoCacheが私のニコニコ動画遍歴を記録し続けています。

えっ！近日公開予定？！

サーバが９代目になりました

Jul 13th, 2008

最強最悪コマンドを発動してしまい、サーバとして機能を失ってしまったのでOSの再インストールを行いました。

本来なら現状復旧の方法を知っておくべき（仕事をしていたらこれが最優先）なのですが、PHPで書かれたブログがMySQLと通信できない不具合が発生していたため、クリーンな状況から始めることにした訳です。

初代の頃はFreeBSDを使っていましたが、CDイメージをダウンロードするのが待ちきれず、細い回線（６４kbps）でFTPを使ってインストールを２〜３時間かけて我慢強くやっていたのを思い出します。

今となれば５０Mbps以上（６４kbpsの７８０倍以上）でる光回線を手にれた私には４GBのDVDイメージなんてコーヒータイムでお釣りがきますw

DVDを使ったインストールは簡単で３０分で終わります♪（FreeBSDは１０分で終わりますww）

今回使用したOSはCentOS5.2です。（OpenSUSE11.0にしようかと一瞬迷ったのは秘密ですw）

＝＝＝＝＝＝

CentOSは何が良いのか？と使うまでずっと疑問でしたが、使ってみてはっきりしました。

まず、RHEL（RedHat Enterprise Linux）の（完全）クローンであることで、RHELで使われているとても安定したアプリケーションや使うことのできるアプリケーションや設定をそのまま流用できること

また、最近バージョン９の配布を始めたFedora Coreの豊富なパッケージ（アプリケーション）も使うことができること

そして、RHEL向けに開発される無償オープンソースのサードパーティ製のアプリケーションがCentOSも動作対象にしていること

さいごに、自宅サーバでいじくり回した成果物を仕事に活かせる可能性もあり、またその逆もあります。

日本市場がWindows系サーバとRHELサーバという（憂うべき）状況であることや多くのエンジニアの人と話しをするのに「RHEL」や「CentOS」というキーワードを使えることが優位であることからCentOSを使うことはいいこと尽くめだと思っていますwww

＝＝＝＝＝＝

５年間でサーバ（OS）は９代目、筐体（PC）は５台目です。

５台目となるDELL製のSC440を５月の上旬に手に入れたのですが、これを会社に回して６台目の購入を視野入れ始めました。

なぜかというとCore2Duo（E6600）は２コアなのでXenで２つのOSしか（実用的に）使えないんです。

そうなると、最近やりたくなったCentOS + Windows XP + Windows Server 2003の３個のOSを同時に動かせません＞＜

＝＝＝＝＝＝

ということで記念すべき１０代目のサーバのスペックは

Quad Core 2.6GHz以上

メモリ４GB以上

が必須となりますw

＝＝＝＝＝＝

といった感じでオタク感を感じる文章を書きましたが、作って楽しいというかベンチマークを取って金儲けに使えないものかと日々企んでいるわけですが、典型的な行動に移すのに時間のかかるタイプなので進捗度１％ですw

そうそう、ブロクが一つ動かなかった件は、「localhost」を「127.0.0.1」にPHPのプログラムを修正してら正常に動くようになりました。

/etc/hostsをいじったのが原因だったようです。
