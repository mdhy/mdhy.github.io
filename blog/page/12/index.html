SSHで証明書を使ってログインする

Jul 22nd, 2009

PuTTYを使ってSSHをしている林です。

最近になって PuTTY Tray という最小化するとタスクトレイに納まってしまうステキなSSHクライアントを使っています。

PuTTY Tray：http://haanstra.eu/putty/

※蛇足ですがPuTTY Portable に入れて使えるようです。

＝＝＝＝＝

PuTTYでぽちぽちすると

login as :

ログインアカウントと求められ、

hoge@hoge.net’s password:

次にパスワードを求められます。

もし間違えると

Access denied

hoge@hoge.net’s password:

とエンドレスです。

＝＝＝＝＝＝

正直、うざいです

＝＝＝＝＝＝

そこでOpenSSHを使って証明書をパスフレーズを付けずに作ってやることにしました。

通常？では

ssh-keygen -t dsa -b 1024 -N -f key

として、パスフレーズを設定するですが

ssh-keygen -t dsa -b 1024 -N ‘’ -f key

として、パスフレーズを空で証明書を作成します。

keyとkey.pubが作成されたら片方をサーバ側（接続される側）に登録します。

cat key.pub >> .ssh/authorized_keys

※必要に応じてsshdの設定を変更する必要があります。CentOSの場合は/etc/ssh/sshd_configなります。

[/etc/ssh/sshd_config]

ServerKeyBits 768 –> ServerKeyBits 1024

PubkeyAuthentication yes

AuthorizedKeysFile .ssh/authorized_keys

↑の２行を追加

SSHクライアント側にはkeyの方を読み込ませるわけですが、私はPuTTYを使っているのでOpenSSHの証明書はそのままでは読み込めません。

どうもppkというPuTTY形式に変換しないといけないようです。

参考にしたサイト

http://tortoisesvn.net/docs/nightly/TortoiseSVN_ja/tsvn-ssh-create-keys.html

＝＝＝＝＝

パスフレーズなしというのは非常に危険な使い方ですが、家庭LANで使うにはさほど問題なさそうです。

これで少しは楽になります。

Ubuntu + Asterisk

Jun 27th, 2009

久しぶりにルーティングについて解らない頭で勉強してみると昔やってたことを思い出しました。

AsteriskでSIPサーバです。

http://ja.wikipedia.org/wiki/Asterisk_(PBX)

懐かしたを覚えて、ちょっと調べてみるとこんなの見つけました↓

http://www.kenti.jp/dat/2006/06/asterisk050.html

おー面白いじゃないか！！とやってみました。

http://voip-info.jp/index.php/Asterisk

↑を参考にソースから入れてみました。

＝＝＝＝＝＝＝

今回の構成は(Hyper-Vで仮想化されています)

・Ubuntu 8.04 Server版

・Asterisk 1.4.25.1（ソース）

Ubuntu Server版はJeOSらしいので何も入っていませんｗ

apt-get -y install gcc g++ libncurses5-dev make patch

↑を実行しておきます。

今回はzaptelを入れないので

./configure —without-zaptel

これでコンパイルは通りました。

＝＝＝＝＝＝＝＝＝

この日のためにso-netフォンを月額８円で契約しましたw

設定したのはsip.confとextensions.confの２つです。

sip.confの内容

[general]

maxexpirey=3600

defaultexpirey=3600

registertimeout=300

context=default

port=5060

bindaddr=0.0.0.0

srvlookup=no

allowguest=no

disallow=all

allow=ulaw

allow=alaw

allow=gsm

language=jp

dtmfmode=inband

nat=never

localnet=192.168.1.0/255.255.255.0

;

;

register => 7:123:0007@192.168.1.10/0123456789

register => 05012345678@so- net.ne.jp:xxxxxxxx:Axxxxxxx@voipxx.so-net.ne.jp/05012345678

;

;

[ntttel]

type=friend

username=0007

secret=123

host=192.168.1.10

canreinvite=no

dtmfmode=inband

disallow=all

allow=ulaw

allow=alaw

context=fromntttel

insecure=very

[iptel]

type=friend

username=Axxxxxxx

secret=xxxxxxxx

host=voipxx.so-net.ne.jp

fromdomain=so-net.ne.jp

fromuser=05012345678

canreinvite=no

dtmfmode=inband

disallow=all

allow=ulaw

allow=alaw

context=fromiptel

insecure=very

;

;

[201]

type=friend

username=201

secret=123

host=dynamic

canreinvite=no

dtmfmode=inband

;

;

extensions.confの内容

[general]

static=yes

writeprotect=yes

[globals]

IPTEL=05012345678

NTTTEL=0123456789

NTTRT=7

[fromiptel]

exten => ${IPTEL},1,Ringing

exten => ${IPTEL},2,Wait,5

exten => ${IPTEL},3,HangUp

[fromntttel]

exten => ${NTTTEL},1,Dial(SIP/201,30)

exten => ${NTTTEL},2,HangUp

;

;

[ipout]

exten => _050.,1,SetCallerId,${IPTEL}

exten => _050.,2,Dial(SIP/${EXTEN}@iptel,60,r)

exten => _050.,3,Congestion

include => nttout

[nttout]

exten => _0.,1,SetCallerId,${NTTRT}

exten => _0.,2,Dial(SIP/${EXTEN}@ntttel,60,r)

exten => _0.,3,Congestion

[dtmf-debug]

exten => s,1,Ringing

exten => s,2,Wait,10

exten => s,3,Answer

exten => s,4,WaitExten(1)

exten => _X,1,SayDigits(${EXTEN})

exten => _X,2,Goto(s,4)

exten => t,1,Goto(s,4)

[default]

include => ipout

include => nttout

include => inner

[inner]

; ip-in-debug

exten => 111,1,Goto(fromiptel,${IPTEL},1)

; Local SIP Telephones 201-208

exten => 201,1,Dial(SIP/201,30,tT)

exten => 201,2,Answer()

exten => 201,3,Voicemail(${EXTEN})

exten => 201,4,Hangup

exten => 201,102,Answer()

exten => 201,103,Voicemail(${EXTEN})

exten => 201,104,Hangup

;

;

＝＝＝＝＝＝

といった感じでVoIPアダプタが無くてもIP電話がつかえるようになりました。

設定は1.2系をコピペしたものですが、動いているので良しとします。
