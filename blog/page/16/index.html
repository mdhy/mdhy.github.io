自宅サーバの悲劇

Feb 7th, 2009

　思い起こせこの前の日曜日。 　おもしろ半分に書いてみたLVM（Logical Volume Maneger）の設定が事故を起こして、システム起動に必要なディレクトリをマウントできずに再起動を無限ループしていました。 　設定を行っているときは、設定が間違っていてエラって無視されるだとうと思っていましたが、よくもわからずカーネルに近い部分をいじるとアバアバしてしまうことを痛感しました。 　＝＝＝＝＝ 　事故を起こした原因は、オンラインでルートディレクトリのあるLV（Logical Volume）にPV（Physical Volume）を適当に追加して何も考えず放置したことだろうと思っています。 　追加したPVをインストールディスクのレスキューモードを使って削除することで復旧しました。 　＝＝＝＝＝ 　今回の事故でデータの損失は無かったのですが、Domain-Uで動いていたWinXPが調子が悪いので、今までのディスクイメージをコピーして新しいディスクイメージを作成してなんてことをやっていると、Domain-0が不安定になるんですよ＞＜ 　仮想ネットワークやらDomain-Uに接続するVNCやらがコケて最終的には再起動しなければいけないことになります。 　なので提供するサービスをDomain-Uで動かしてバックアップやサービスの拡張の際に土台であるDomain-0に影響をあたえないようにすることにしました。 　OpenVPNと内部DNSをUbuntu810Serverで動かして、WEBサービスとメールサービスをFreeBSD6.4で動かすことにしています。 　＝＝＝＝＝＝ 　Unbuntuのシンプルさには惚れました♪ 　FreeBSD6.4は懐かしいですが、Xen上でディスクアクセスが遅いです＞＜原因は分かっていません；； 　なんてことをしていたらもう一週間ですよ。 　さて止まってるサービスを再構築しないといけない。

LVM on Software-RAID

Jan 25th, 2009

　『MEZZO』『BECK』というアニメーション（略してアニメｗ）をみている林です。 　一日中エアコンを元でアニメを観ていると喉が痛くなってきました。 　社員旅行ではインフルエンザの保菌者として扱われていましたが現実になりそうです。 　＝＝＝＝＝ 　火を落とした旧サーバから取り外した８０GBのHDD（ハードディスク）を新サーバに増設しました。 　構築の予定から増設を決めていたのでLVMで構築してあります。 　加えて当初の計画とおりに 　（これまでHDDが故障してアボーン＞＜となったことのない私としては,定期的にバックアップを取っておけば十分でしょって考えだったんですが） 　定期バックアップが面倒なのとディスクが飛んで復旧するのが面倒（たぶん諦めるｗ）なのでLinuxのソフトウェアRAIDを使ってRAID-0を構成しています。 　＝＝＝＝＝ 　LVMを構築するとなにがよいかというと（以下参照） 　参考URL：http://www.itmedia.co.jp/enterprise/0307/11/epn01.html 　要するに（雑に説明すると）物理ドライブを組み合わせて柔軟に『領域』を作れたり、スナップショットというバックアップもとれるということです。 　今回使っているのは、柔軟に領域を操れるというところです。 　参考URL：http://centossrv.com/centos5-raid-1.shtml 　現在はCentOS５を使っているので、グラフィカルで設定を進めてしまったので説明は省きますが、今回の作業を備忘録します。 　＝＝＝＝＝ 　したいこと 　８０GBのHDDを新しく追加して、初期の構成で使ってなかった１６０GBの半分とRAID-0を構成して、それを初期の構成のLVMのVGに追加してLVに追加する。 　手順 　１．追加した８０GBのHDDにパーティションを作成する。 　　fdiskを使って基本領域を１つ作って、領域タイプを「Linux raid 自動検出」にしておく。（詳細は省く） 　　（１６０GBのHDDの空きにも同じ操作をする） 　２．RAID-0でデバイスを作成する。 　　# mdadm —create /dev/md2 —level=1 —raid-devices=2 /dev/sda3 /dev/sdc1 　　# mdadm —detail —scan | grep md15 >> /etc/mdadm.conf 　　（再起動してもマウントされるように設定しておく） 　　参考URL：http://beer.dip.jp/puki/index.php?RAID1%2F%B9%BD%C3%DB(mdadm) 　３．PV(Physical Volume)に登録する。 　　# pvcreate -v /dev/md2 　　# pvscan 　　(登録状況を確認する) 　４．既存のVG(Volume Group)にPVを登録する。 　　# vgextend -v VolGroup10 /dev/md2 　　# vgdisplay -v 　　（登録状況を確認する） 　５．LV(Logical Volume)を作成する。 　　# lvcreate -L 74.5G -n LogVol12 VolGroup10 　　参考URL：http://d.hatena.ne.jp/oragoku/20070727/1185548488 　６．あとはext3にフォーマットする。 　　# mke2fs -j /dev/VolGroup10/LogVol12 　＝＝＝＝＝ 　おまけ 　VGに追加したPVを取り出したい場合 　　# vgreduce -v VolGroup10 /dev/md2 　　参考URL：http://www.itmedia.co.jp/enterprise/0308/08/epn01.html
