SSHで証明書を使ってログインする

Jul 22nd, 2009

　PuTTYを使ってSSHをしている林です。 　最近になって PuTTY Tray という最小化するとタスクトレイに納まってしまうステキなSSHクライアントを使っています。 　PuTTY Tray　：　http://haanstra.eu/putty/ 　※蛇足ですがPuTTY Portable に入れて使えるようです。 　＝＝＝＝＝ 　PuTTYでぽちぽちすると 　login as : 　ログインアカウントと求められ、 　hoge@hoge.net&squot;s password: 　次にパスワードを求められます。 　もし間違えると 　Access denied 　hoge@hoge.net&squot;s password: 　とエンドレスです。 　＝＝＝＝＝＝ 　正直、うざいです 　＝＝＝＝＝＝ 　そこでOpenSSHを使って証明書をパスフレーズを付けずに作ってやることにしました。 　通常？では 　# ssh-keygen -t dsa -b 1024 -N -f key 　として、パスフレーズを設定するですが 　# ssh-keygen -t dsa -b 1024 -N &squot;&squot; -f key 　として、パスフレーズを空で証明書を作成します。 　keyとkey.pubが作成されたら片方をサーバ側（接続される側）に登録します。 　# cat key.pub >> .ssh/authorized_keys 　※必要に応じてsshdの設定を変更する必要があります。CentOSの場合は/etc/ssh/sshd_configなります。 >　[/etc/ssh/sshd_config] > >　#ServerKeyBits 768 –> ServerKeyBits 1024 > >　PubkeyAuthentication yes > >　AuthorizedKeysFile .ssh/authorized_keys > >　↑の２行を追加 　SSHクライアント側にはkeyの方を読み込ませるわけですが、私はPuTTYを使っているのでOpenSSHの証明書はそのままでは読み込めません。 　どうもppkというPuTTY形式に変換しないといけないようです。 　参考にしたサイト 　http://tortoisesvn.net/docs/nightly/TortoiseSVN_ja/tsvn-ssh-create-keys.html 　＝＝＝＝＝ 　パスフレーズなしというのは非常に危険な使い方ですが、家庭LANで使うにはさほど問題なさそうです。 　これで少しは楽になります。
