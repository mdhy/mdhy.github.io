W2k8 + Utorrent ってどうよ

Sep 18th, 2009

　こんばんは。 　週末の旅行に備えて洗濯とか冷蔵庫の整理とかをしている林です。 　世の中にはBitTorrent（ビットトレント）という技術があって良しも悪しも世界中で使われています。 　私も度々インストールし直して使うことがあるんですが、暇を持て余してまたインストールした話です。 　＝＝＝＝＝＝＝ 　今まで何度も書いてきたようにHyper-V Server 2008 R2を使って3つほどOSを常時稼動させているんですが、リソース（資源）をあまらせています。 　（あっ洗濯1回目終わった） 　そこで4つ目をインストールしてみようと思い立ち、以前にも使った事のあるuTorrentを使うことにしたのはいいのですがOSは何にしようかと考えるわけです。 　見つけましたよ。Windows Server 2008 R2 RCｗ 　RC（製品候補版）でも1年は使えるはずなのでインストールを始めてWindows Server 2008 Enterprise Server-Coreを選んで30分くらいでインストールは完了しました。 　Windows Server 2008になってから何が困ったちゃんかというとファイアウォールが固めてあって思うように外部と通信ができないことです。 　さらにこれがServer-Coreとなるとリモートデスクトップを有効にするのでさえ >　# cscript c:\windows\syste32\scregedit.wsf /ar 0 　上記のようにコマンドプロンプトから操作を行う必要があります。 　まぁログインするとコマンドプロンプトがひとつしか出てないし、いろいろと使えないものがたくさんありましたｗ 　外部の端末からMMC管理を行えばいいようなんですが、「アクセスが拒否されました」の繰り返しでうまくいかないので断念しました＞＜ 　＝＝＝＝＝＝ 　次にutorrentのインストールを行うわけですがコマンドプロンプトからutorrent.exeを実行してやればすんなりインストール完了です。 　torrentクライアントの機能として正常に稼動していますが、フォルダダイアログは正常に動作してませんでした。 　まぁもうちょっと使ってみないとわからないんですが・・・様子見ですね。
