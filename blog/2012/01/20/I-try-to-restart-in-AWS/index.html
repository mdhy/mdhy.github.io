AWS で 再開してみる

Jan 20th, 2012

あけましておめでとうございます。 去年の夏頃から更新をしていませんでした。 林です。 このブログを家から出して運用しようとずーと考えていました。 Google App Engine で動かそうと頑張ってみたものの、Java の上で PHP を動かそうとか考えていたのですが、 file_get_contents() が 使えないことがわかり放置してました。 その後、AWS (Amazon Web Service) の Amazon EC2 で動かして見ようとマクロインスタンスを作成しようとしたところ、オール英語でうーんで放置。 そうだ！何年か前から契約しているさくらのレンタルサーバがあることを思い出したが BASIC 認証が使えないだよなーで放置。 そして、つい数日前に AWS、「Windows Server」を無料使用枠に追加 : http://japan.cnet.com/news/service/35013142/ が出てきたー！！ Linux に Apache (ligttpd) + php + MySQL の構成なんで何十台分作ったので、変わった構成でこのブログ動かしたいという気持ちがふつふつ沸いてきてここ（AWS）に引っ越してきましたーw このブログが動くまでなにしたか以下にかいていきます。 ### 情報収集 Amazon Web Services ブログ ; http://aws.typepad.com/aws_japan/ ### アカウント作成 AWS Management Console : http://aws.amazon.com/jp/console/ とりあえず、上にあるサインアップをクリックして必要事項を入力していく。 ※クレジットカードが必要 ### インスタンス作成 AWS Management Console を用いたインスタンスの生成 : http://d.hatena.ne.jp/nishio-dens/20110326/1301133309 ぱっとみよく分からない設定があるので見ようみまねで設定していく。 東京リージョンでマイクロインスタンスの Windows Server 2008 R2 Datacenter Edition でインスタンスを作成する。 Elastic IP をインスタンスにひもづける（無料） ### インスタンスに接続 Windows なので 3389/tcp でアクセス ### Web サーバ環境を構築 XAMPP を IE でダウンロードしてインストール、結構はやい http://www.apachefriends.org/jp/xampp-windows.html ファイルのアップロードには freeftpd で SFTP をつかう FreeFTPDでSSL通信対応のFTPサーバ構築 : http://iziz.blog.shinobi.jp/Entry/291/ ### 公開完了 PHP のバージョンが上がっているのでエラーログ全開なので細かい修正をして設置完了 ### 感想 日本語になって AWS Management Console も使いやすいし、1 年間月額無料はおいしい これからこのブログをいじりながら AWS であぞんでみようとおもう。 そして寝る
