SQLって面白いし使いやすい

Oct 13th, 2008

youtubeの再生リストをフルスクリーンで表示できるソフトウェアを作ってみた林です。

今回は二つのことを書きます。

＝＝＝＝＝＝

一つ目はいまさらyoutubeの新しい使い方を見つけたことについて。

使った開発環境はVisual Basic 2005 Express Edition(http://www.microsoft.com/japan/msdn/vstudio/express/past/2005/)です。

いつ使っても使いやすいIDE（統合開発環境）ですよ。みなさんつかってますかー

作りはテキストボックスにyutubeの再生リストのURL「http://jp.youtube.com/view_play_list?p=xxxxxxxxxxxxxxxx%E3%80%8D%E3%81%AEx%E9%83%A8%E3%82%92%E5%85%A5%E5%8A%9B%E3%81%97%E3%81%A6%E3%83%9C%E3%82%BF%E3%83%B3%E3%82%92%E6%8A%BC%E3%81%99%E3%81%A8%E5%A4%96%E9%83%A8%E3%83%97%E3%83%AC%E3%82%A4%E3%83%A4%E3%83%BC%E3%81%AEURL%E3%81%A7%E3%81%82%E3%82%8B%E3%80%8Chttp://jp.youtube.com/swf/cps.swf?datatype=playlist&eurl=&data=%E3%80%8D%E3%81%A8%E7%B5%90%E5%90%88%E3%81%97%E3%81%A6%E3%80%81%E3%83%96%E3%83%A9%E3%82%A6%E3%82%B6%E3%82%B3%E3%83%B3%E3%83%9D%E3%83%BC%E3%83%8D%E3%83%B3%E3%83%88%E3%81%A7%E8%A1%A8%E7%A4%BA%E3%81%99%E3%82%8B%E3%81%A0%E3%81%91%E3%81%A7%E3%81%99%E3%80%82

フルスクリーンに引き延ばすと画質は悪いんですが、んまー楽しめます。

フルスクリーン以外でも使えるので、自身で再生リストを作成して、エンドレスBGMを流すのも簡単に実現できます♪

同じようなことやって様々な実装を紹介しているサイトはたくさんありますー

＝＝＝＝＝

もうひとつはSQLでけっこうなことができることについて。

SQLってRDBMSで使われるデータの操作や定義を行う言語のことです。

参考URL：http://ja.wikipedia.org/wiki/SQL

独学で勉強を進めていると「SELECT A,B FROM DATA_TABLE」とかを軸に考えるんですが、あれに触れてしまった私は今回の試練を乗り越えました。

あれ：SQLクイックリファレンス

ブログのトピックを月ごとに集計して月降順に並べようというものです。↓のように。

2008年10月（１０）

2008年7月（５）

2008年５月（２）

2008年1月（６）

2007年12月（２）

2007年7月（５）

経過を説明しなければいけないんですが、いろいろと端折ると

「SELECT YEAR,MONTH,COUNT(*) AS C FROM TOPICTABLE GROUP BY MONTH OEDER BY MONTH DESC」

と書いたら↓のようになってしまい、

2008年５月（２）

2008年10月（１０）

2008年1月（６）

2007年12月（２）

2007年7月（１０）

浅知恵で修正して

「SELECT CONCAT(YAER,MONTH) AS YM,YEAR,MONTH,COUNT(*) AS C FROM TOPICTABLE GROUP BY YM ORDER BY YM DESC」

としたら↓になり、

2008年7月（５）

2008年５月（２）

2008年10月（１０）

2008年1月（６）

2007年12月（２）

2007年7月（５）

あれちゃんとソートできてないよって感じで、あれを思い出して↓に至るわけですw

「SELECT CONCAT(YEAR,CASE CHAR_LENGTH(MONTH) WHEN 1 THEN CONCAT(‘0’,MONTH) WHEN 2 THEN MONTH END) AS YM,YEAR,MONTH,COUNT(*) AS C FROM gb_main GROUP BY YM ORDER BY YM DESC」

人間が理解してソートするように一発関数がなかったので、1,2,3という１文字だったら0を追加して01,02,03となるように制御フロー関数を使ってみました。

参考URL：http://dev.mysql.com/doc/refman/5.1/ja/index.html

やはり公式のリファレンスが役にたちますよ♪
