MySQLに移行完了しました

Dec 23rd, 2008

　日曜日にスゴい風でまったく寝付けずに昨日は１２時間くらい寝てしまった林です。 　睡眠は人生を無駄にしていると言われますが、たしかにそのとおりでしょうｗ 　＝＝＝＝＝＝ 　このブログのデータ（PostgreSQLに格納されている）をMySQLに移すことが簡単に実現できたことを以前の記事で紹介しました。 　以前の記事：http://www.medanohaya.net/?p=384 　次はこのブログのデータベースとの接続部分をMySQLと通信できるように変えないといけません；； 　ここが面倒くさいなーーーということなんですが、このブロクを最後に書いたのは今年の３月くらいでした。 　たしか、PostgreSQLでもMySQLでもすぐにセットアップできるように設計したこと思い出して、データベース関連のソースを見てみると、それっぽく書いてありました。（書いたのは私自身なんですが；；） 　過去の記憶を手探りしながら、一日コースだな＞＜なんて思いながら、手を加えてみると以下のようになりました。 function sql_connect($IN_USER=&squot;&squot;,$IN_PASS=&squot;&squot;,$IN_DBNAME=&squot;&squot;,$IN_HOST=&squot;localhost&squot;,$IN_PORT=&squot;5432&squot;){ 　$db_my = new PDO(“mysql:host=$IN_HOST;dbname=$IN_DBNAME”,$IN_USER,$IN_PASS); 　$db_my->query(&squot;SET NAMES utf8&squot;); 　return $db_my; // return pg_connect(“user=$IN_USER password=$IN_PASS dbname=$IN_DBNAME host=$IN_HOST port=$IN_PORT”); } function sql_close($IN_CON){ 　$IN_CON = null; 　return true; // return pg_close($IN_CON); } function sql_query($IN_CON, $IN_STRING){ 　return $IN_CON->query($IN_STRING); // return pg_query($IN_CON, $IN_STRING); } function sql_fetch_row($IN_OBJECT){ 　return $IN_OBJECT->fetch(PDO::FETCH_BOTH); // return pg_fetch_row($IN_OBJECT); } function sql_fetch_array($IN_OBJECT){ 　return $IN_OBJECT->fetchall(PDO::FETCH_BOTH); // return pg_fetch_all($IN_OBJECT); } function sql_num_rows($IN_OBJECT){ 　return $IN_OBJECT->rowCount(); // return pg_num_rows($IN_OBJECT); } 　だけ直せば問題ないようでした。 　ただし、SELECT文でカラム名を大文字で指定するとPDO::FETCH_BOTHの配列の結果も大文字で返ってくるため、不具合がでるところも数カ所ありました。 　が、小文字で指定してやれば小文字で返ってきたので問題ありませんでした。 　＝＝＝＝＝＝ 　これで、無事に移行ができそうです。 　来年を予定してましたが、気持ちよく年を越せそうです♪
